# Google-Search-Analysis
📊 Google Search Analysis Using Pytrends

A Data Science project that analyzes Google search trends using Python and the Pytrends library to extract insights about keyword popularity, regional demand, and time-based trends.

📌 Project Overview

In this project, I worked as a data-driven analyst to extract and analyze Google search behavior.

The objective was to answer business-focused questions such as:

How can we dynamically analyze multiple keywords?

Which countries search for a specific keyword the most?

How does search interest vary globally?

How has keyword interest changed over time?

How do related keywords compare?

This project demonstrates real-world data extraction, analysis, and visualization techniques.

🚀 Key Features & Analysis Performed
1️⃣ Dynamic Keyword Search Function

Created reusable Python code where only the keyword needs to be changed to analyze different search terms.

This makes the project scalable and efficient for marketing and product research.

2️⃣ Top 15 Countries Analysis

Extracted country-wise search interest.

Identified the top 15 countries with the highest search volume.

Visualized using bar charts.

📌 Business Use Case:
Helps companies identify high-demand markets for expansion.

3️⃣ World Map Visualization

Built an interactive choropleth map using Plotly.

Highlighted countries based on search intensity.

📌 Business Use Case:
Enables strategic geographic targeting.

4️⃣ Time-wise Trend Analysis

Extracted interest_over_time() data.

Plotted trends across years/months.

Identified spikes and seasonal patterns.

📌 Business Use Case:
Supports demand forecasting and campaign timing.

5️⃣ Keyword Comparison

Compared multiple related keywords.

Plotted comparative trend graphs.

📌 Business Use Case:
Useful for competitor analysis and SEO strategy.

⚠️ Important Note About Pytrends

This project uses Pytrends, which is an unofficial Google Trends API.

Since it is not officially supported by Google:

Sometimes API requests may fail.

Temporary errors may occur.

If an error appears, simply re-run the cell — it usually works after retrying.

These issues are due to request limits or temporary blocking, not code errors.
